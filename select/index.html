# SQL风格查询

除了集合风格查询外，sqala还支持类似SQL风格的查询DSL，首先我们需要做一些准备：

import sqala.dsl.*

val people = asTable[People]

我们需要使用asTable方法将实体类转换为表结构，以便后续查询中使用。

# 构建查询

sqala的查询使用query from方法构建，from的参数是使用asTable创建的表结构：

val q = query from people

查询返回的数据类型为：

val result: List[People] = db.fetch(q)

# 投影

select方法用于生成数据投影，其参数是表达式 (opens new window)或其组成的元组、命名元组：

val q = query from people select (id = people.id, name = people.name)

查询返回的数据为：

val result: List[(id: Int, name: String)] = db.fetch(q)

使用selectDistinct对结果集去重：

val q = query from people selectDistinct (id = people.id, name = people.name)

# 过滤

使用where方法进行条件过滤，参数是一个Boolean表达式 (opens new window)：

val id = 1
val q = query from people where people.id == id select (id = people.id, name = people.name)

多次调用where时将会使用AND来连接查询条件。

对于动态拼接的条件，sqala提供了whereIf方法，会在第一个参数值为true时使用AND才将条件拼接到查询中：

val id = 1
val name = "小黑"

val q = (query from people).whereIf(id > 0)(people.id == id).whereIf(name.nonEmpty)(people.name == name)

filter不会改变查询结果类型。

# 表连接

使用join、leftJoin、rightJoin来处理表连接，并使用on来处理连接条件，并将其放入from方法中：

val q =
    query from (a join b on a.id == b.id join c on b.id == c.id)

此时返回的结果类型为：

val result: List[(A, B, C)] = db.fetch(q)

由于外连接会产生额外的空值，因此，sqala可以从表连接的信息中自动推断返回类型，如果我们把上面的查询改为先进行右连接再进行左连接：

val q =
    query from (a rightJoin b on a.id == b.id leftJoin c on b.id == c.id)

此时返回的结果类型变成了：

val result: List[(Option[A], B, Option[C])] = db.fetch(q)

这里显式写出了返回类型仅仅是为了演示，sqala会从查询中自动推断出合适的返回类型。

对于自连接的情况，我们使用as方法给表起别名，比如有一个存储部门树的表对象：

case class Department(
    id: Int,
    parentId: Int,
    name: String
)
val department = asTable[Department]

我们可以这样来关联查询：

val d1 = department as "d1"
val d2 = department as "d2"
val q = query from (d1 leftJoin d2 on d1.parentId = d2.id)

请注意，在同一个查询中，出现的表别名不能重复，否则会产生编译错误。

# 分组

使用groupBy配合聚合函数表达式进行分组操作：

val q = (
    query
        from people
        groupBy (people.name, people.gender)
        select (name = people.name, gender = people.gender, count = count(), maxBirthday = max(people.birthday))
)

另外可以使用having过滤聚合出的数据：

val q = (
    query
        from people
        groupBy (people.name, people.gender)
        having count() > 0
        select (name = people.name, gender = people.gender, count = count(), maxBirthday = max(people.birthday))
)

# 排序

使用orderBy方法进行排序，参数是使用asc或desc方法生成的排序规则：

val q = query from people orderBy (people.id.asc, people.name.desc)

sortBy不会改变查询结果类型

# 子查询

sqala支持表达式 (opens new window)中的子查询和JOIN中的表子查询，首先来介绍表达式中的子查询：

返回一列数据的子查询可以作为表达式的一部分出现，比如：

val subQuery = query from people select max(p.birthday)

val q = query from people where people.birthday > subQuery

支持IN、ANY、SOME、ALL、EXISTS等子查询相关谓词：

val subQuery = query from people where people.name == "小黑" select people.id

val q1 = query from people where people.id == any(subQuery)

val q2 = query from people where people.id.in(subQuery)

使用as给查询起别名之后，我们可以将其放入from的表中：

val t1 = query from b select (x = b.x, y = b.y) as "t1"

val q = query from (a leftJoin t1 on a.x == t1.x) select (ay = a.y, by = t1.y)

作为表的子查询必须将其投影成命名元组（NamedTuple），使用命名元组（NamedTuple）给字段起名之后，就可以在后续使用字段名引用对应的字段。

# 集合操作

sqala支持使用union、unionAll、intersect、intersectAll、except、exceptAll等方法来处理集合查询，此类操作的使用方法与集合风格查询类似，不再赘述。
