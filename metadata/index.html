# 元数据配置

使用sqala构造查询、查询结果映射之前，我们需要配置好sqala的元数据信息。

# 基础配置

sqala的实体使用Scala的case class进行管理，比如我们有一个people表，建表语句（以MySQL为例）如下：

CREATE TABLE `people` (
    `id` int NOT NULL AUTO_INCREMENT COMMENT '主键',
    `name` varchar(50) NOT NULL COMMENT '姓名',
    `gender` tinyint(1) NOT NULL COMMENT '性别',
    `birthday` date NOT NULL COMMENT '生日',
    `id_number` varchar(18) NULL COMMENT '身份证号',
    PRIMARY KEY (`id`)
);

我们可以写出这样的实体类：

import java.util.Date

case class People(
    id: Int, 
    name: String, 
    gender: Int, 
    birthday: Date,
    idNumber: Option[String]
)

实体类字段的类型和表字段的类型一一对应，可空字段使用Scala标准库的Option类型，实体类字段命名采用Scala标准的驼峰命名规则，sqala将会在生成查询时将其转换为使用_拼接的蛇形命名规则。

这样，我们就可以使用sqala来构建查询了：

import sqala.dsl.*

val q = query[People].filter(p => p.id == 1)

# 详细配置

上面的例子里，我们在数据库中把id字段设置为了主键，为了让sqala知晓如何正确生成查询，我们需要使用primaryKey注解标记主键字段：

import sqala.dsl.*
import sqala.dsl.annotation.*

import java.util.Date

case class People(
    @primaryKey id: Int, 
    name: String, 
    gender: Int, 
    birthday: Date,
    idNumber: Option[String]
)

这样，就可以使用对象创建以主键字段更新其他字段的语句：

val people = People(1, "小明", 1, Date(), None)

val u = update(people)

另外，id字段除了是主键字段之外，还是一个自增字段，我们可以使用autoInc注解来标记这个字段：

import sqala.dsl.*
import sqala.dsl.annotation.*

import java.util.Date

case class People(
    @autoInc id: Int, 
    name: String, 
    gender: Int, 
    birthday: Date,
    idNumber: Option[String]
)

这样，sqala就可以在使用实体对象插入数据时，在生成的SQL中忽略掉该字段，同时也可以使用实体对象，用主键字段更新其他字段：

val people = People(1, "小明", 1, Date(), None)

// 跳过id字段
val i = insert(people)

// 使用id字段更新其他字段
val u = update(people)

另外，如果实体类的字段名和表的字段名不符合驼峰命名到蛇形命名的映射规则的话，也可以使用column注解指定实际的字段名：

import sqala.compiletime.*
import sqala.compiletime.annotation.*

import java.util.Date

case class People(
    @autoInc @column("people_id") id: Int, 
    name: String, 
    gender: Int, 
    birthday: Date,
    idNumber: Option[String]
)

sqala支持在实体类上添加如下注解：

注解名 目标 作用
@table 类 指定表名
@primaryKey 字段 把字段标记为主键
@autoInc 字段 把字段标记为自增主键
@column 字段 指定字段名

# 自定义字段类型

sqala内置支持的字段类型有：

字段类型 字段类型
scala.Int scala.String
scala.Long scala.Boolean
scala.Float java.util.Date
scala.Double java.time.LocalDate
scala.math.BigDecimal java.time.LocalDateTime

以及这些类型对应的Option类型。

上面的例子里，我们使用了Int类型来处理性别字段，这样做安全性比较差，其可能在生成查询时传入预期之外的值，更好的做法是使用Scala3提供的enum：

enum Gender:
    case Male
    case Female
    case Other

然后我们修改一下之前的实体类定义，却会得到一个编译错误：

case class People(
    @autoInc id: Int, 
    name: String, 
    gender: Gender, 
    birthday: Date,
    idNumber: Option[String]
)

// 编译错误
val q = query[People].filter(p => p.gender == Gender.Male)

原因是sqala无法处理自定义的类型，为其生成查询，为了达成这一目标，需要使用Scala3的given功能给自定义类型实现CustomField这个trait。如果很多实体类都用到了同一个自定义类型，为了让编译器在没有显式导入的情况下也知道如何合成代码，推荐将这个实现放入其伴生对象中：

import sqala.dsl.*

enum Gender:
    case Male
    case Female
    case Other

object Gender:
    given CustomField[Gender, Int] with
        override def fromValue(x: Int): Gender = x match
            case 1 => Gender.Male
            case 2 => Gender.Female
            case _ => Gender.Other

        override def toValue(x: Gender): Int = x match
            case Gender.Male => 1
            case Gender.Female => 2
            case Gender.Other => 3

CustomField有两个类型参数，第一个是要支持的自定义类型，第二个是sqala支持的内置类型。

我们需要实现两个方法，fromValue定义如何进行反序列化操作，toValue定义如何生成查询。

这样，就可以在查询时可以使用自定义的类型了：

val q = query[People].filter(p => p.gender == Gender.Male)
