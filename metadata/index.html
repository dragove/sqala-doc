# 元数据配置

在使用sqala构造查询之前，我们需要配置好sqala需要的元数据信息。

# 引入依赖

在使用之前，确保引入sqala的依赖：

libraryDependencies += "com.wz7982" % "sqala-jdbc_3" % "latest.integration"

# 基础配置

sqala使用Scala的case class创建查询。

比如，我们有如下people表，建表语句（以MySQL为例）如下：

CREATE TABLE `people` (
    `id` int NOT NULL AUTO_INCREMENT COMMENT '主键',
    `name` varchar(50) NOT NULL COMMENT '姓名',
    `gender` tinyint(1) NOT NULL COMMENT '性别',
    `birthday` date NOT NULL COMMENT '生日',
    `id_number` varchar(18) NULL COMMENT '身份证号',
    PRIMARY KEY (`id`)
);

其可以对应到下面的case class：

case class People(
    id: Int,
    name: String,
    gender: Int,
    birthday: LocalDate,
    idNumber: Option[String]
)

case class的字段名和类型要与数据库表结构对应，可空字段使用Option类型接收。

case class的类名和字段名使用标准的驼峰规则命名，sqala将会在生成SQL时将其映射到蛇形命名。

这样，我们就可以使用sqala来构建查询了：

import sqala.static.dsl.*

val q = queryContext:
    query[People].filter(p => p.id == 1)

# 设置主键

上面的表结构中，id字段是主键，为了让sqala知晓如何正确生成查询，我们使用primaryKey注解标记主键字段：

import sqala.static.dsl.*

case class People(
    @primaryKey id: Int,
    name: String,
    gender: Int,
    birthday: LocalDate,
    idNumber: Option[String]
)

这样，就可以使用对象创建以主键字段更新其他字段的语句：

val people = People(1, "小明", 1, LocalDate.now(), None)

val u = update(people)

我们也可以使用autoInc注解标记自增主键：

import sqala.static.dsl.*

case class People(
    @autoInc id: Int,
    name: String,
    gender: Int,
    birthday: LocalDate,
    idNumber: Option[String]
)

这样，在使用对象生成插入语句时，将会自动跳过主键字段：

val people = People(1, "小明", 1, LocalDate.now(), None)

// 自动跳过id字段
val i = insert(people)

# 自定义名称

如果实体类名和字段名不符合驼峰风格到蛇形风格的映射规则的话，我们可以使用table和column注解标记实际的名称：

import sqala.static.dsl.*

@table("t_a")
case class A(
    @column("col_1") x: Int
)

# 自定义字段类型

上面的例子中，我们使用Int类型来接收性别字段，但这样做安全性比较差，其可能生成预期之外的值，更好的做法是使用enum：

enum Gender:
    case Male
    case Female
    case Other

我们把实体类定义改为：

case class People(
    @autoInc id: Int,
    name: String,
    gender: Gender,
    birthday: LocalDate,
    idNumber: Option[String]
)

但在创建查询时：

// 编译错误
val q = queryContext:
    query[People]

会产生一个编译错误，其原因是sqala并不知道这样的自定义类型如何处理为SQL表达式，所以我们需要做一些额外的工作。

我们需要使用given为自定义类型实现trait CustomField，推荐将实现放在类型的伴生对象中：

import sqala.static.dsl.*

enum Gender:
    case Male
    case Female
    case Other

object Gender:
    given CustomField[Gender, Int] with
        override def fromValue(x: Int): Gender = x match
            case 1 => Gender.Male
            case 2 => Gender.Female
            case _ => Gender.Other

        override def toValue(x: Gender): Int = x match
            case Gender.Male => 1
            case Gender.Female => 2
            case Gender.Other => 3

Custom的两个类型参数分别为要实现的类型，和要映射到的基础类型；fromValue方法处理如何从基础类型创建自定义类型的值，toValue方法含义反之。

这样，在创建查询时就可以使用自定义的类型了：

import sqala.static.dsl.*

val q = queryContext:
    query[People].filter(p => p.gender == Gender.Male)

sqala内置支持的字段类型有：

字段类型 字段类型
scala.Int scala.String
scala.Long scala.Boolean
scala.Float java.time.LocalDate
scala.Double java.time.LocalDateTime
scala.math.BigDecimal 以上类型对应的Option类型
sqala.dsl.Json

需要使用其他字段类型时均需要使用CustomField将其映射到内置支持的类型。
